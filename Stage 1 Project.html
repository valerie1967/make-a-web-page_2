Lesson 1 Basic Web and HTML concepts


HTML


HTML, which stands for Hyper Text Markup Language, creates the look of the text and other parts of web pages through the use of elements which are created with tags. Here is a link to a website that has a great tutorial on HTML 5.

TAGS

Tags can be closed or not, this means that the Text or URL reference is surrounded by a greater than sign a letter or group of letters denoting the type of element then a less than sign and then the text and then the close to the element which consists of a greater than sign, a slash and the same letter or group of letters and another less than sign.
Example:<p>

Elements

Elements come in two flavors either inline or Block, the difference being the Block elements encase the text or what have you in a box, while the inline elements do not.

Lesson 2 Creating a Structured Document with HTML


CSS: A First Look

CSS can define elements on the HTML page changing things like font size and color. This is done to avoid repetition. It is easier to define a style for an element and apply is to all like elements, thanks to individually apply it to each DIV or H1 Tag.

Boxes, Boxes and More Boxes

All parts of a web page are in boxes.Even the ones that seem NOT box-like or rectangularly shaped are in boxes. Boxes are easily re-arranged on web pages.

Tree-like Structure

Element of HTML can be nested inside one another creating a Tree-like Structure. This type of structure is defined by a parent child relationsip between Tag elements where a Child element will be within it's parent counter-part.

Text Editors

There are many types of text editors that developers use when creating the code for HTML web pages. Scratchpad, Codepen and Sublime are only three of them. They have varying degrees of ease of use, the ability to share and the languages they can accommodate. For example, ScratchPad is easy to use but can't be shared, Codepen can be shared but you need an account. Sublime can be used with HTML, CSS and Java but is not beginner friendly.

Lesson 3 CSS Cascading Style Sheets


CSS: A Deeper Look

CSS or Cascading Style Sheets Control the way the elements look in HTML. Here is a link to a website that has a great tutorial on CSS. CSS can be written for a specific element or for a "class". It can be specified on the html document, either in the Header or inline with the element. It can also be specified in a separate CSS document and linked or referred to in the HTML with the link tag.
The syntax would be as follows:


Element, curly brace, property: value; curly brace

Example:
p {
color: green;
}

For a Class

Example:
< div class="nameofclass"> </div>
<style>
.nameofclass {
color: green;
}
</style>

To link an CSS file with an HTML file

Example:
In the Head Element
<link rel="stylesheet" type="text/css" href="nameofcssfile.css">

An advantage of using CSS is the avoidance of repetition. You can define the look of specific elements once and apply the defined aspects to all like parts once. If they don’t look right then you only have to troubleshoot the smaller CSS definition and not an entire HTML document.
